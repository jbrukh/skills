Introducing Web Deck

Browser-native presentations for CoinFund

What Is Web Deck

Single HTML file
Every deck is one self-contained file. No build step, no npm, no framework. Open it in any browser.
CoinFund brand system
Cream backgrounds, Nunito typography, semantic color palette. Matches the house style pixel-for-pixel.
Static or dynamic
Choose print-optimized static mode or interactive dynamic mode with keyboard nav, transitions, and touch support.
Git-versionable
Each slide is a <section> block. Diffs are clean, merges are meaningful, history is useful.

Live Development Workflow

  1. Create deck.html Start from the boilerplate, add slides using 5 content patterns.
  2. Start the server Run serve-deck.mjs. It watches the file and auto-reloads the browser on every save.
  3. Iterate Edit slides with the Edit tool. Changes appear instantly. No manual refresh.
  4. Export PDF Click "Download as PDF" in the overlay menu. Puppeteer renders pixel-perfect 16:9 pages.

The overlay menu and live-reload client are injected at serve time. The deck.html on disk stays clean and portable.

Features at a Glance

Navigation Arrow keys, Space, Page Up/Down, touch/swipe, hash deep links
Animations Staggered fade-up reveals via .reveal class, respects prefers-reduced-motion
Overview mode Press P to see all slides at once, click any to jump
PDF export Server-side Puppeteer rendering, one-click download from overlay menu
Content patterns Header+body blocks, label-value grids, tables, numbered lists, callout boxes
Press F for fullscreen. Press Esc to exit.
